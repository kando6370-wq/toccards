## 使用说明

- 以下 5 项需产品 / 研发 / 法务侧拍板或提供凭证，否则对应模块只能用 Mock 推进、无法上线。
- 卡牌基础数据源已改为当前 D1 中的 `cards_all` / `games` / `sets` / `tcgplayer_skus`，不再需要额外卡牌接口厂商或卡牌接口凭证；采集程序写入节奏归入研发内部联调项。
- 每项在开发阶段均可用 Mock / 测试凭证推进，**不阻塞 M0 工程骨架启动**；但 P0 项越早确定，越能减少后期适配返工。
- 其余「技术实现阶段确认」类 TBD（TTL、类型共享工具、Admin Token 存储等）属研发内部决策，不在本清单。

## 🔴 P0 — 阻塞核心功能实现，请优先确认

### 1. 汇率接口提供方
- **需确认**：选定汇率数据源（如 open exchange / 厂商自带 / 其他）及接入凭证
- **为何阻塞**：Home / Collection / 卡牌详情的多币种金额换算展示
- **决策方**：产品 / 研发
- **当前状态**：可 Mock 汇率推进，上线前须接入

---

## 🟡 P1 — 阻塞账号与通知，开发中期需到位

### 2. 邮件服务（验证码发送）
- **需确认**：Resend 或 AWS SES 二选一，开通账号 + API Key + 发信域名
- **为何阻塞**：邮箱注册验证码、找回密码验证码的真实发送
- **决策方**：研发 / 运营
- **当前状态**：可 Mock 发送推进

### 3. Apple Sign In 凭证
- **需确认**：Apple 开发者账号下的 Service ID / Team ID / Key ID（及私钥）
- **为何阻塞**：Apple 登录（iOS 上架强制要求）
- **决策方**：研发（依赖 Apple 开发者账号归属方）

### 4. Google OAuth 凭证
- **需确认**：Google Cloud 项目的 OAuth Client ID / Client Secret
- **为何阻塞**：Google 登录
- **决策方**：研发

---

## 🟢 P2 — 上架前必须，可后置

### 5. 法务与商店链接
- **需确认**：服务条款 URL（`terms_url`）、隐私政策 URL（`privacy_url`）、App Store 链接（`app_store_url`）的实际地址
- **为何阻塞**：Profile 内协议链接、分享 App 功能；App Store 审核必需项
- **决策方**：产品 / 法务
- **当前状态**：写入 `app_config` 即生效，M8 上线前必须填入
