二、计算涨幅
1. 通用涨跌幅计算 = (当前价格 - 周期起点的市场价) / 周期起点的市场价 × 100%
2. 周期价格取值：
7D Change % = (当前价格 - 7 天前价格) / 7 天前价格 × 100%
30D Change % = (当前价格 - 30 天前价格) / 30 天前价格 × 100%
3M 6M:   周期 Change % = (当前价格 - 周期起点价格) / 周期起点价格 × 100%
说明：
周期起点价格取该周期起点当天的有效市场价。
如果周期起点当天无价格，优先使用该时间点之前最近一次有效价格。
如果没有任何有效历史价格，则涨跌金额展示 --，涨跌幅展示 -/-。

3. Portfolio 总资产涨跌计算 = (当前文件夹资产价值 - 周期起点时该文件夹总资产) / 周期起点时该文件夹总资产 × 100%
4. Collection Item 当前价值计算=当前市场价格*数量
单条 Collection Item 涨跌幅= (当前市场价格 - 周期起点时市场价格) / 周期起点时市场价格 × 100%
说明：
     a  涨跌幅按单张价格变化计算。
Quantity 只影响涨跌金额，不影响涨跌百分比。
因此同一张卡 Qty 1 和 Qty 3 的涨跌百分比相同，但涨跌金额不同。
5. Home Most Valuable 涨跌= (当前单卡市场价 - 30D前单卡市场价) / 30D前单卡市场价 × 100%
说明：
Most Valuable 按单张当前市场价排序。
Quantity 不影响 Most Valuable 排序。
Quantity 不影响 Most Valuable 的涨跌幅。
如果需要展示涨跌金额，展示单张涨跌金额，不展示总持仓涨跌金额。
如果 30 天前无有效价格，涨跌幅展示 -/-。

6. Search 列表涨跌=  (Current Market Price - 30D Previous Market Price) / 30D Previous Market Price × 100%

十四、用户资产与游客状态说明

规则定位

App 支持游客状态使用。用户未注册 / 未登录时，也可以在 App 内进行主要功能操作，并产生本地游客资产。

游客资产包括但不限于：

1. Portfolio 文件夹；
2. Collection Item；
3. Wishlist；
4. 扫描添加记录；
5. Search 快捷收藏记录；
6. 货币偏好；
7. 金额隐藏偏好；
8. 文件夹排序和默认文件夹设置；
9. 其他与用户收藏资产相关的本地数据。

⸻

游客状态可用功能

游客状态下，用户可以正常使用 App 内主要功能，包括：

1. 浏览 Home；
2. 使用 Search；
3. 使用 Scan；
4. 添加卡牌到 Portfolio；
5. 添加卡牌到 Wishlist；
6. 新建 / 编辑 / 删除 Portfolio 文件夹；
7. 编辑 Collection Item；
8. 查看卡牌详情；
9. 切换货币；
10. 隐藏 / 显示金额；
11. 进行其他不强依赖账号的本地操作。

游客状态下产生的数据先绑定到本地游客身份。

⸻

游客身份定义

游客身份是 App 在本地为未登录用户生成的临时身份。

规则：

1. 用户首次打开 App 且未登录时，系统生成一个本地游客 ID。
2. 游客 ID 用于关联游客状态下产生的 Portfolio、Wishlist、Collection Item 等资产数据。
3. 游客资产仅在当前设备本地可用。
4. 游客资产不具备跨设备同步能力。
5. 用户卸载 App、清除本地数据或更换设备后，游客资产可能丢失。
6. 引导注册时可提示用户注册账号以保存资产。

⸻

游客状态下注册账号

场景

用户当前处于游客状态，并且已经产生了一些游客资产。此时用户选择注册新账号，并注册成功。

处理规则

游客状态下注册成功后，需要将当前游客资产迁移到注册成功的账号下。

迁移范围包括：

1. Portfolio 文件夹；
2. Collection Item；
3. Wishlist；
4. 扫描添加记录；
5. 文件夹排序；
6. 默认文件夹；
7. 货币偏好；
8. 金额隐藏偏好；
9. 其他本地游客资产相关配置。

迁移结果

1. 游客资产迁移到新注册账号。
2. 注册成功后，用户身份切换为已登录状态。
3. Home、Collection、Wishlist、Search Qty、Collected 状态等页面刷新为新账号资产数据。
4. 原游客资产不再作为独立游客资产重复展示。
5. 后续新增 / 编辑 / 删除资产均归属该账号。
6. 用户之后在其他设备登录该账号时，可同步查看已迁移资产。

PRD 规则

游客状态下注册成功账号后，系统将当前设备上的游客资产迁移到该新注册账号下。迁移成功后，用户进入登录态，后续所有资产操作均归属该账号。

⸻

游客状态下登录已有账号

场景

用户当前处于游客状态，并且已经产生了一些游客资产。此时用户选择登录一个已有账号。

处理规则

游客状态下登录已有账号时，不自动迁移游客资产。

规则：

1. 登录成功后，App 展示该已有账号的云端资产。
2. 当前游客资产保持不变，但不自动合并到该账号。
3. 登录后，Home、Collection、Wishlist、Search Qty、Collected 状态刷新为已登录账号的数据。
4. 游客资产不覆盖账号资产。
5. 账号资产不合并游客资产。
6. 后续新增 / 编辑 / 删除资产均归属当前登录账号。

PRD 规则

游客状态下登录已有账号时，不自动迁移当前游客资产。登录成功后，App 切换为该账号资产数据，游客资产保持不变，不与已有账号资产合并。

⸻

注册与登录的区别

场景	游客资产处理	登录后展示
游客注册新账号	游客资产迁移到新账号	展示迁移后的新账号资产
游客登录已有账号	游客资产不迁移、不合并	展示已有账号资产

核心区别：

1. 注册新账号：默认认为用户希望把当前游客资产保存到新账号，因此迁移游客资产。
2. 登录已有账号：默认认为用户要进入已有账号环境，因此不自动合并游客资产，避免污染或覆盖已有账号资产。

⸻

为什么登录已有账号不自动迁移

登录已有账号时不自动迁移游客资产，是为了避免以下问题：

1. 用户只是临时登录账号，不一定希望合并当前设备游客资产。
2. 已有账号可能已经有 Portfolio、Wishlist、文件夹结构，自动合并会造成重复资产。
3. 同一张卡可能在游客资产和账号资产中同时存在，自动合并容易产生重复 Collection Item。
4. 游客资产可能来自他人设备或临时体验，不能默认归属到该账号。
5. 自动合并后难以撤销，风险较高。

因此，登录已有账号时，只切换到已有账号资产，不自动迁移游客资产。

退出登录后的处理

用户从已登录状态退出登录后：

1. 当前账号资产不删除。
2. 本地展示切回游客状态。
3. 如果设备上仍存在未迁移游客资产，则展示该游客资产。

5. 不允许继续展示刚退出账号的资产数据。
6. 再次登录账号后，重新展示该账号资产。

删除账号后的处理

用户删除账号后：

1. 账号资产按删除账号规则处理。
2. App 切换为游客状态。
3. 不将已删除账号资产回退为游客资产。
4. 如果本地存在旧游客资产，按游客身份继续展示；否则展示空游客状态。
