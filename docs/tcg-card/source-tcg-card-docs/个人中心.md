

Profile 模块 PRD
一、页面定位
Profile 是用户管理账号、恢复购买、联系客服、评分、分享 App、查看协议与退出登录的页面。
首版不做订阅，因此页面中 Upgrade to Pro / Unlock All / Go unlock / Subscribe 等订阅相关区域不进入 1.0 需求，需要从首版页面中删除或隐藏。

⸻

二、页面入口
用户点击底部导航 Profile 进入。
游客进入 Profile 后展示游客态页面。
已登录用户进入 Profile 后展示登录态页面。
用户点击账号区域后进入 Account 账号详情页。
用户点击 Customer Support 后进入客户支持页。
用户点击 Terms of Use / Privacy Policy 后跳转官网查看协议。

3.2 规则

注册成功后，用户卡牌资产转移到当前账号上。登录成功后游客的卡牌资产不迁移。

8.2 Type 规则
Type 支持多选。
用户可选择一个或多个反馈类型。
再次点击已选项可取消选择。
如果用户未选择 Type，页面未体现是否必填。建议 Type 可选，未选时按 Other 处理，降低提交门槛。
8.3 Function 规则
Function 支持多选。
用户可选择一个或多个相关功能模块。
再次点击已选项可取消选择。
首版不做订阅，Subscription 选项应删除或隐藏。
如果用户未选择 Function，页面未体现是否必填。建议 Function 可选，未选时按 Other 处理。
8.4 Email 规则
Email 为必填项。
用户必须输入有效邮箱后才可提交。
邮箱格式需要校验。
无效邮箱不允许提交。
已登录用户进入客服页时，Email 默认填入当前账号邮箱。
游客用户进入客服页时，Email 为空，需要用户手动填写。
用户可修改默认邮箱。
邮箱错误提示
空邮箱：
Please enter your email.
邮箱格式错误：
Please enter a valid email address.
8.5 Message 规则
Message 为反馈内容。
页面未明确是否必填。建议 Message 必填，否则客服无法处理。
Message 为空时不允许提交。
Message 建议限制 1000 字符。
超长时提示：Message must be 1000 characters or less.
Message 错误提示
Please enter your feedback.
8.6 Submit Feedback 规则
点击 Submit Feedback 后校验 Email 和 Message。
校验失败时，在对应字段展示错误提示。
校验通过后提交反馈。
提交中按钮展示 loading，并禁止重复点击。
提交成功后展示成功提示。
提交失败后保留用户填写内容，并展示失败提示。
提交成功后是否清空表单页面未体现，建议清空表单并返回 Profile 或停留当前页展示成功状态，需要确认。
成功提示
Feedback submitted. Thank you.
失败提示
Unable to submit feedback. Please try again later.

九、Score 评分
9.1 入口
Profile 页面点击 Score。
9.2 规则
点击 Score 后，优先调用系统原生评分弹窗。
如果当前系统允许展示原生评分弹窗，则直接调起原生评分弹窗。
如果原生评分弹窗调用次数已用尽，或系统未展示弹窗，则跳转 App Store 写评论页面。
用户关闭评分弹窗后返回 Profile。
用户从 App Store 返回后回到 App。
如果跳转失败，留在原页面
9.3 说明
iOS 原生评分弹窗存在系统展示频率限制，不能保证每次点击都出现。
所以按钮策略应为：
优先原生评分弹窗；
次级兜底跳转 App Store 评论页。

十、Share With Friends 分享 App
10.1 入口
Profile 页面点击 Share With Friends。
10.2 规则
点击 Share With Friends 后，调起苹果原生分享弹窗组件。
分享内容为 App Store 下载链接。
用户可分享到第三方 App。
用户取消分享后，不提示错误，停留 Profile。

10.3 分享内容
分享内容：
app store中app的下载链接

十二、Log Out 退出登录
12.1 入口
Profile 已登录页底部 Log Out。
Account 详情页中的 Log Out。
12.2 规则
点击 Log Out 后退出当前账号。
退出成功后回到游客态 Profile。
退出登录不删除账号。
退出登录不删除该账号绑定的云端卡牌资产。
退出后，Portfolio、Wishlist、账号详情等账号资产数据不再展示。
用户再次登录同一账号后，恢复该账号绑定的资产。
退出失败时提示：Unable to complete this action. Please try again later.
【补充】退出登录前可展示确认弹窗，避免误触；页面当前未画确认弹窗。

⸻

十三、账号与卡牌资产绑定规则
用户的 Portfolio、Wishlist、Collection Item、扫描添加记录、文件夹等数据与账号绑定。
已登录状态下新增、编辑、删除的卡牌资产都归属于当前账号。
切换账号后，只展示新账号下的数据。
退出登录后，Profile 回到游客态。
游客状态注册成功则把游客数据转移到账号上，此时退出登录变成一个无任何数据的游客。游客状态如果登录成功则游客的数据还是在游客上，此时退出登录恢复游客数据。
