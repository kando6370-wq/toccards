卡牌详情 / Portfolio Collection Item PRD

一、页面定位

卡牌详情页用于展示单个收藏对象的基础信息、价格趋势、市场价格、交易入口，以及用户已收藏时的 Collection Item 信息。

收藏对象包括：

1. TCG 单卡；
2. 体育卡；
3. 评级卡；
4. 套盒 / 卡包 / 整箱；
5. 其他特殊收藏品。

该页面存在两种状态：

1. 未加入 Portfolio 的详情页

用户从 Search、Scan 匹配结果、Wishlist、市场卡牌列表进入。

页面展示：

1. 基础信息；
2. Price；
3. Market Prices；
4. Shop 交易入口。

不展示用户持有信息。

2. 已加入 Portfolio 的详情页

用户从 Collection - Portfolio 列表、Home - Most Valuable、扫描添加成功后的 Portfolio 记录进入。

页面展示：

1. 基础信息；
2. Price；
3. Shop；
4. Collection Item 信息。

支持：

1. 编辑 Collection Item；
2. Remove from Portfolio；
3. 分享。

⸻

二、页面入口

未加入 Portfolio 详情入口

1. Search 列表点击卡牌 / 产品。
2. Wishlist 列表点击卡牌 / 产品。
3. Trending Today 点击卡牌 / 产品。
4. Shop / Marketplace 场景中点击卡牌 / 产品进入。

已加入 Portfolio 详情入口

1. Collection - Portfolio 列表点击卡牌 / 产品。
2. Home - Most Valuable 点击卡牌 / 产品。
3. 扫描添加成功后，从 Portfolio 列表进入。
4. Portfolio 卡牌详情页编辑后返回当前详情。

七、Collection Item 字段

Collection Item 展示用户对当前对象的持有记录。不同对象类型字段有所不同，但尽量复用统一结构。

本次调整不修改 Price Tab 规则。

1. 图表切换：RAW、GRADED。
2. 图表时间周期：1M、3M、6M、12M、MAX。

6. Price 图表、Market Prices、Shop 可分区加载。

9.4 编辑规则

1. 点击已加入 Portfolio 详情页中的 Edit item 进入编辑状态。
2. 编辑页顶部基础信息保持展示，不因编辑状态变化。
3. 用户点击 Cancel 后放弃修改，返回详情页展示原数据。
4. 用户点击 Save changes 后保存修改。
5. 保存成功后返回已加入 Portfolio 详情页，并刷新 Collection Item 展示。
6. 保存失败时停留编辑页，用户已输入内容不得丢失。
7. 保存成功后，Home、Collection、Most Valuable、图表等涉及该对象的数据需要按新字段刷新。

⸻

9.5 Quantity 规则

1. Quantity 必填。
2. Quantity 必须为正整数。
3. Quantity 不能为 0。
4. 修改 Quantity 后，该 Collection Item 当前价值随之变化。
5. 当前价值 = 对应市场价 × Quantity。
6. Quantity 修改后，Home 总资产需要同步更新。

⸻

9.6 Portfolio 规则

1. Portfolio 字段展示当前所属文件夹。
2. 用户可切换 Portfolio 文件夹。
3. 如果用户将对象移动到其他文件夹，保存后该对象从原文件夹移除，加入新文件夹。
4. 文件夹变更后：
    * 原文件夹总资产刷新；
    * 新文件夹总资产刷新；
    * Home / Collection 如果当前正在查看原文件夹，该对象不再展示；
    * 如果当前查看新文件夹，该对象应展示在列表中。
5. 如果保存失败，不改变原文件夹归属。

⸻

9.7 Grader / Condition / Grade 规则

1. Grader 用于决定该 Collection Item 的价格取值口径。
2. Raw 表示未评级。
3. PSA / BGS / SGC / TAG / CGC / AGS 表示评级对象。
4. 用户选择 Raw 时，价格取 Raw / Ungraded 市场价。
5. 用户选择评级机构时，必须选择对应等级。
6. 评级机构和等级共同决定 Graded 市场价。
7. 如果所选评级机构 / 等级无市场价，当前价值展示 --，但允许保存。
8. Condition 用于 Raw 品相。
9. 用户选择评级机构后，Condition 隐藏或置灰，避免 Raw 品相和评级状态混用。

⸻

9.8 Purchase Price 规则

1. Purchase Price 是用户记录的购买成本。
2. Purchase Price 不参与当前市场价值计算。
3. Purchase Price 可为 0。
4. Purchase Price 按当前 App 货币展示。
5. 如果切换货币，Purchase Price 需要换算。

⸻

9.9 Notes 规则

1. Notes 为用户备注。
2. Notes 可为空。
3. 保存后展示在 Collection Item 详情中。
4. 字数上限页面未体现，限制 500 字符。

10.1 Remove from Portfolio

Remove from Portfolio

规则

1. 点击 Remove from Portfolio 后，展示二次确认弹窗。
2. 用户确认后，将当前 Collection Item 从 Portfolio 移除。
3. 移除成功后，该对象不再出现在对应 Portfolio 文件夹中。
4. 移除成功后，Home 总资产、Home 图表、Most Valuable、Collection 列表刷新。
5. 如果当前对象仅有这一条 Collection Item，移除后返回 Collection Portfolio 列表。
6. 如果同一对象还有其他 Collection Item，移除当前项后返回上一页或刷新详情，需确认。
7. Remove from Portfolio 不影响 Wishlist 状态。

10.2 Remove from Wishlist

未加入 Portfolio 且已加入 Wishlist 的详情页，底部展示：

Remove from Wishlist

规则

1. 点击 Remove from Wishlist 后，展示二次确认弹窗。
2. 用户确认后，将当前对象从 Wishlist 移除。
3. 移除成功后，该对象不再出现在 Wishlist 列表中。
4. Remove from Wishlist 不影响 Portfolio 状态。

十二、数据与展示规则

12.1 金额规则

1. 所有金额按 App 当前选择货币展示。
2. 切换货币后，详情页金额同步换算。
3. 金额保留 2 位小数。
4. 百分比不因货币切换变化。
5. 价格缺失展示 --。

12.2 Price 与 Collection Item 的关系

1. Price Tab 展示公共市场价格。
2. Collection Item 展示用户持有记录。
3. 用户在 Collection Item 中选择 Raw / Graded、Condition、Language、Finish 等字段后，会影响该持有项的价格取值。
4. Home 总资产、Collection 当前价值、Most Valuable 都应使用 Collection Item 中保存后的字段取价。
5. Purchase Price 不参与市场价值，只用于用户成本记录。

12.3 Raw / Graded / Sealed 显示规则

1. Raw 对象展示 Condition。
2. Graded 对象展示 Grader + Grade。
3. Sealed Product 展示 Status = Sealed。
4. Raw 品相和评级状态不可混用。
5. Sealed Product 不展示 Raw / Graded 作为主状态。
